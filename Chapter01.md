# ネットワーク学習環境の構築
Linuxのネットワークについて学習をするための環境を構築します。本教科書の実習では、以下のような環境を想定しています。

・ホストOSとしてWindows 11が動作するPC
一般的なWindows PCを想定していますが、VirtualBoxが動作するmacOS、Linuxでも可能です。

・有線LAN、あるいはWi-Fiでインターネットに接続可能
インターネットに接続するためのルーターがあり、実習用のPCが有線LAN、またはWi-Fiで接続されていて、インターネットに接続できます。

実習用のPCはDHCPを使って、IPアドレスが自動的に設定されています。

・Linuxが動作する仮想マシン
VirtualBoxで仮想マシンを動作させ、AlmaLinux 9.7をゲストOSとして動作させます。

仮想マシン実行環境はVirtualBox以外のものでも構いませんが、ネットワーク設定はそれぞれの仮想マシンソフトウェアによって異なるので、機能や設定方法は個別に確認してください。

# インターネット接続用ルーター
実習環境は、インターネット接続用のルーターが存在し、そのルーターを通じてインターネットに接続できることを想定しています。

想定しているルーターは、一般の家庭で使用するWi-Fiのアクセスポイントを兼ねたものです。Wi-Fiの接続の他、スイッチングハブ機能を内蔵し、有線LANでも接続が可能です。

ルーターはDHCPサーバー機能も備えており、接続したPCやスマートフォンなどにIPアドレスやデフォルトゲートウェイ、名前解決のためのDNSサーバーの情報などを提供します。

以後、「ルーター」と単純に記述した場合、この家庭用Wi-Fiルーターを指します。

## 企業内ネットワークとの違い
企業など数多くのクライアントがネットワークに接続する環境の場合、インターネット接続用のルーター、Wi-Fiのアクセスポイント、DHCPサーバーなどの機能はそれぞれ別々の機器を使用して構成することが多くなります。

家庭用ルーターではすべての機能がオールインワンで備わっていますが、ネットワークにどのような機能が必要なのか理解する必要があります。

# スイッチングハブ
有線LANでPCを相互に接続する場合、スイッチングハブが必要です。

接続速度は1Gbpsが主流ですが、最近では2.5Gbpsや10Gbpsといった高速なネットワークを構築できるものも増えています。

以後、「スイッチ」と単純に記述した場合、スイッチングハブのことを指します。

※スイッチングハブの図を入れる

## スイッチはEthernet接続
スイッチはEthernetと呼ばれるプロトコルで相互に接続します。

Ethernetは「OSI 7階層」で下から1番目と2番目の層に当たるため、「L2スイッチ」と呼ばれることがあります。

※OSI7階層の図を入れる

スイッチの中には、IPによるルーティングの機能を内蔵したものもあり、「L3スイッチ」と呼ばれます。

## カスケード接続
ルーターにはスイッチ機能が備えられていますが、ポート数が限られています。より沢山のPCを有線LAN接続する場合には、別途スイッチを用意する必要があります。

スイッチとスイッチを相互に接続することを「カスケード接続」と呼びます。カスケード接続すると、1つのポート数の多いスイッチとして機能するようになります。

# Wi-Fi接続
Wi-Fi接続は、有線LANのようにケーブルの接続が不要なため、PCやスマートフォンをネットワークに接続するために多く使われるようになりました。

## EthernetとWi-Fiの関係
Wi-Fiは、「IEEE 802.11」という国際標準規格に則った無線LANのことを指します。

有線LANで使っているEthernetは「IEEE 802.3」として規格化されており、どちらも同じL2ネットワークを担っています。そのため、簡単に変換が可能なので、有線LANと無線LANで相互に接続が可能です。

## Wi-Fiのセキュリティ機能
実習では、同じネットワークに接続されている別のPCから仮想マシン上のゲストOSにアクセスすることがあります。

一部のWi-Fiルーターには、セキュリティを高めるため接続しているPCの相互接続を許さない設定がされている場合があります。

実習で別のPCから仮想マシンにアクセスできない場合、この機能が有効になっていないか、ルーターの管理画面で確認してみてください。

# VirtualBoxで使用できるネットワーク
VirtualBoxでは、様々な接続の仮想ネットワークが利用できます。

大きく分けて、ホスト以外の外部のコンピューターと接続する「外部接続」ネットワークと、「仮想マシンのみ」、「仮想マシンとホストのみ」の3種類に分類できます。

外部接続は、さらにIPによる「NAT」接続と、イーサネットによる「ブリッジ」接続に分けられます。

以下、主に使用するネットワーク接続について解説します。これら以外のネットワークについては説明は割愛します。

## NAT接続
仮想マシンの設定では「NAT」という選択肢になっています。

ゲストOSからの外部への接続をホストOSからの接続に変換します。変換はIPアドレスとTCP/UDPのポート番号を使用しているので、厳密にいえば「NAPT」（Network Address Port Translation）を行っています。

※NATの図を入れる

## ブリッジ接続
仮想マシンの設定では「ブリッジアダプター」という選択肢になっています。

仮想マシンからの外部への接続をホストマシンのネットワークインターフェースにEtnernetで接続します。Ethernetのレベルで相互に接続することを「ブリッジ」と呼びます。

※ブリッジの図を入れる

## 仮想マシンのみ接続
仮想マシンの設定では「内部ネットワーク」という選択肢になっています。

仮想マシンのみを相互に接続する仮想スイッチに接続します。
外部に一切接続できないため、動作が他のPCに影響したり、セキュリティ的に問題があったりする検証を行う際などに使用します。

※内部ネットワークの図を入れる

## ホストオンリー接続
仮想マシンの設定では「ホストオンリーアダプター」という選択肢になっています。

仮想マシンとホストOSに用意されたホストオンリーアダプターを相互に接続する仮想スイッチに接続します。

# ブリッジ接続を試してみる
上記の接続のうち、これまで使用していないブリッジ接続を試してみましょう。

## ブリッジ接続の前提条件
ブリッジ接続を試すには、以下の前提条件を想定しています。

- ホストPCが接続しているネットワークでDHCPサーバーが動作しており、ホストOSもDHCPクライアントとしてIPアドレスを取得している
- 仮想マシンのNAT接続でゲストOSがインターネットに接続できている

## 実習のシナリオ
この状態で、仮想マシンのネットワークアダプター1を「NAT」から「ブリッジアダプター」に変更してみます。ゲストOSのネットワークインターフェースに、外部で動作しているDHCPサーバーからIPアドレスが設定されることを確認します。

## NAT接続時のIPアドレスの確認
NAT接続を行っている時のIPアドレスを確認します。ゲストOSでipコマンドを使って確認します。

VirtualBoxのNAT接続は、デフォルトでVirtualBoxの用意したDHCPサーバー機能が動作しており、ゲストOSに対して「10.0.2.15」のIPアドレスを設定するようになっています。

## ブリッジ接続への変更
仮想マシンのネットワークアダプターの設定を「NAT」から「ブリッジアダプター」に変更します。手順は以下の通りです。

1. Oracle VirtualBox マネージャーで「仮想マシン」を選択します。
2. 使用している仮想マシンを選択します。
3. 「設定」ボタンをクリックします。
4. 「ネットワーク」を選択します。
5. 「アダプター1」タブの割り当てを「NAT」から「ブリッジアダプター」に変更します。
6. 「OK」をクリックします。

ネットワークアダプターの設定変更は即時有効になります。

## ブリッジ接続時のIPアドレスの確認

外部のDHCPサーバーからIPアドレスが設定されていることが確認できます。

# VirtualBoxのネットワーク設定
VirtualBoxは、デフォルトでホストOS側に「ホストオンリーアダプター」という仮想ネットワークインターフェースを作成して「192.168.56.1」のIPアドレスを設定します。そしてホストオンリーに接続した仮想マシンに対してDHCPサーバーでIPアドレスを自動設定できるようにします。

この設定が正しく行われていなかったり、IPアドレスを変更したりするなどのネットワーク設定を行うには、VirtualBoxの環境設定で高度な設定が行えるように変更する必要があります。

## VirtualBoxの高度な設定の有効化
VirtualBoxの高度な設定を有効化するには、VirtualBoxの環境設定画面を呼び出し、ウインドウ左上の「高度」をクリックし、「OK」ボタンをクリックします。

※SS

Oracle VirtualBox マネージャーの左側の選択肢が増えていることが確認できます。

## ネットワーク設定の確認
高度な設定が行えるようになると、Oracle VirtualBox マネージャーで「ネットワーク」が選択できるようになります。

ネットワークの設定画面では、「ホストオンリーネットワーク」、「NATネットワーク」、「クラウドネットワーク」の設定が行えます。

ここでは、ホストオンリーネットワークの設定を確認してみます。

# ホストオンリーネットワークの確認
「ホストオンリーネットワーク」タブを選択します。リストに現在設定されているホストオンリー接続のためのネットワークが表示されます。

## ホストオンリーアダプターのIPアドレス
リストからネットワークを選択すると、画面下の「アダプター」タブに、そのホストオンリーネットワークに接続されているホストOS側のホストオンリーアダプターのIPアドレス設定が表示されます。

IPアドレスを変更したい場合には、ここで変更を行います。


## ホストオンリーネットワークのDHCPサーバー
「DHCPサーバー」タブでは、そのホストオンリーネットワークに接続された仮想マシンに対してIPアドレスの自動設定を行うためのDHCPサーバーのパラメータが設定できます。

ホストオンリーアダプターのIPアドレスを変更した場合には、それに合わせてこちらも変更する必要があります。

サーバーアドレスはDHCPサーバー自体のIPアドレスです。ホストオンリーアダプターに割り当てられたIPアドレスの次のアドレスが指定されます。

アドレス下限からアドレス上限の範囲が、仮想マシンに対して自動設定されるIPアドレスの範囲です。

## 固定IPアドレスの範囲を設定
仮想マシンに固定IPアドレスを割り当てたい場合には、DHCPサーバーのアドレス下限または上限を変更して、固定IPアドレスとして手動で割り当て可能なIPアドレスの範囲を設定する必要があります。

たとえば、アドレス下限を「192.168.56.100」に設定すると、「192.168.56.3〜192.168.56.99」はDHCPで自動的に設定されなくなります。

# Wiresharkによるパケットキャプチャ
ネットワークを流れている通信は、パケットという単位でやり取りが行われています。通常はパケットを見ることはできませんが、パケットキャプチャを行うことで見ることができます。

パケットキャプチャツールのWiresharkをインストールして、パケットキャプチャを行ってみましょう。

## Wiresharkのインストール
dnfコマンドを実行して、Wiresharkをインストールします。

$ sudo dnf install wireshark

## wiresharkグループへの所属設定
Wiresharkを実行するユーザーはwiresharkグループに所属している必要があります。

usermodコマンドを実行して、ユーザーのサブグループにwiresharkグループを追加します。サブグループの追加なので、usermodコマンドに-aオプションをつけて実行してください。

$ sudo usermod -a -G wireshark linuc

## グループ所属設定の反映
グループ所属の設定をGUI環境に反映させるために、システム再起動を行います。

再起動後、ログインしてidコマンドを実行してユーザーのサブグループにwiresharkグループが追加されたことを確認します。

```
$ id
uid=1000(linuc) gid=1000(linuc) groups=1000(linuc),10(wheel),984(wireshark) context=unconfined_u:unconfined_r:unconfined_t:s0-s0:c0.c1023
```

## Wiresharkの実行
Wiresharkを実行します。

GUIから実行する場合には、画面左上の「アクティビティ」をクリックし、画面下のDockの一番右の「アプリケーションを表示する」をクリックして、Wiresharkのアイコンをクリックします。

コマンドラインから実行する場合には、wiresharkコマンドを実行します。

```
$ wireshark &
```

## インターフェースの指定
パケットキャプチャを行うネットワークインターフェースを指定します。

ここでは、NAT接続でインターネットと通信ができるenp0s3を選択します。

画面左上の「パケットキャプチャを開始します」ボタンをクリックします。

## 通信を行ってパケットをキャプチャする
指定したインターフェースを使ってネットワーク通信を行い、パケットをキャプチャします。

特別何もしないでも、バックグラウンドで様々なプロセスが通信を行っていますが、Webブラウザを起動してインターネット上のサイトにアクセスしてみます。

ある程度パケットキャプチャが行えたら、画面左上の「パケットキャプチャを停止します」ボタンをクリックします。

## IPアドレスを指定したフィルタリング
パケットキャプチャの結果は非常に多くのパケットをキャプチャしているので、必要となるパケットのみに絞り込むためフィルタを設定します。

「表示フィルタ」と書かれたテキスト入力欄に以下の通りに入力すると、送信元がNAT接続しているIPアドレスのパケットのみに絞り込まれます。

```
ip.src=10.0.2.15
```

## ポート番号を指定したフィルタリング
フィルタに以下の通りに入力すると、宛先のTCPポートが443番、HTTPSプロトコルの通信パケットのみに絞り込まれます。

```
tcp.dstport==443
```

## 複数の条件を指定したフィルタリング
上記2つの条件を同時に指定したい場合には、2つの条件を「&&」でつなげて指定します。

```
ip.src=10.0.2.15 && tcp.dstport==443
```
